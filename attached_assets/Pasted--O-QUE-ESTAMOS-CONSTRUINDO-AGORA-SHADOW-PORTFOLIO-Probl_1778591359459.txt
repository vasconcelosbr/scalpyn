## O QUE ESTAMOS CONSTRUINDO AGORA: SHADOW PORTFOLIO

### Problema
Quando um ALLOW da L3 não vira trade por falta de saldo, some sem rastro.
Isso causa dois problemas:
1. O XGBoost treina apenas com trades executados — dataset com viés de seleção de capital
2. O usuário não sabe quanto dinheiro "deixou na mesa" por falta de saldo

### Solução: Shadow Portfolio
Quando não há saldo para executar, o sistema:
1. Cria um "shadow trade" — simula a compra com U$ 1.000 USDT fixos
2. Acompanha o ativo via OHLCV histórico até TP, SL ou timeout
3. Registra o resultado em `shadow_trades` (nova tabela)
4. Grava também em `trade_simulations` — alimenta o dataset do XGBoost automaticamente
5. Expõe os dados na aba "Shadow Trade" da UI

### Regra absoluta de separação
Shadow data NUNCA contamina:
- P&L real
- Win rate real
- Capital em uso
- Posições abertas
- Qualquer métrica de performance real

Shadow data SÓ aparece em:
- Tabela `shadow_trades`
- Registros em `trade_simulations` com `is_simulated=TRUE`
- Endpoints `/api/shadow-trades/*`
- Aba "Shadow Trade" na UI
- Dataset de treinamento do XGBoost

### Parâmetros do shadow trade
- **Valor investido**: U$ 1.000 USDT (fixo por enquanto)
- **Entry price**: open da próxima candle 1m após o ALLOW
- **TP/SL**: sempre os mesmos do config real do usuário no momento do ALLOW
- **Convenção intra-candle**: SL verificado antes de TP (conservador)
- **Timeout**: `timeout_candles` do config real do usuário

## FLUXO COMPLETO DO SHADOW PORTFOLIO

```
L3 ALLOW gravado em decisions_log
    ↓
execute_buy.py — gate de capital (SpotCapitalManager)
    ├── TEM SALDO
    │       ↓
    │   Executa ordem real na corretora (fluxo existente — não mudar)
    │
    └── NÃO TEM SALDO (ou outro gate: max_positions, circuit_breaker, cooldown...)
            ↓
        ShadowTradeService.create_from_skip(decision, skip_reason, user_config)
        [fire-and-forget — nunca bloqueia o fluxo real]
            ↓
        Cria registro em shadow_trades:
          status = PENDING
          amount_usdt = 1000.0
          entry_price = open da próxima candle 1m
          tp_price = entry_price * (1 + tp_pct/100)
          sl_price = entry_price * (1 - sl_pct/100)
          features_snapshot = decisions_log.metrics.indicators_snapshot
          config_snapshot = config real do momento
            ↓
        [5 minutos depois — Celery task shadow_trade_monitor]
            ↓
        Para cada shadow trade PENDING/RUNNING:
          Busca candles 1m após entry
          Verifica SL antes de TP em cada candle
          Quando resolve: status = COMPLETED
            ↓
        ShadowTradeService.record_as_simulation(shadow_trade)
            ↓
        Grava em trade_simulations:
          is_simulated = True
          decision_type = 'ALLOW'
          result = WIN | LOSS | TIMEOUT
          [compatível com DatasetBuilder.load_simulations()]
            ↓
        DatasetBuilder já inclui automaticamente no próximo treino
```
