## PROBLEMA 1 — Worker Structural: RuntimeError + PendingRollbackError

### Evidência nos logs (recorrente, mesmo após restart)

```
RuntimeError: Event loop is closed
  asyncpg/protocol/protocol.pyx:650
    self._cancellations.add(self._loop.create_task(self._cancel(waiter)))
  asyncio/base_events.py:545
    raise RuntimeError('Event loop is closed')

→ sqlalchemy.exc.PendingRollbackError: Can't reconnect until invalid
  transaction is rolled back.
  app/tasks/collect_market_data.py:323 → run_db_task → session.begin()
```

### Causa raiz confirmada

Todas as tasks Celery usam este pattern em `_run_async`:

```python
def _run_async(coro):
    loop = asyncio.new_event_loop()
    try:
        return loop.run_until_complete(coro)
    finally:
        loop.close()  # ← fecha antes do asyncpg terminar cleanup
```

O `loop.close()` é chamado enquanto o asyncpg ainda precisa de
`loop.create_task()` para fechar a conexão NullPool corretamente.

### Passo 1 — Localizar todos os arquivos com _run_async

```bash

```

Reportar: lista de arquivos e número de linha de cada ocorrência.

### Passo 2 — Verificar se o fix já foi aplicado em algum arquivo

```bash

grep -A 12 "def _run_async" backend/app/tasks/compute_indicators.py
grep -A 12 "def _run_async" backend/app/tasks/compute_scores.py
grep -A 12 "def _run_async" backend/app/tasks/evaluate_signals.py
```

Reportar: o bloco `pending = asyncio.all_tasks(loop)` existe ou não

### Passo 3 — Aplicar o fix em todos os arquivos que ainda não têm

Substituir em **cada arquivo** que não tem o fix:

```python
# ANTES:
def _run_async(coro):
    """Run async code in sync Celery task."""
    loop = asyncio.new_event_loop()
    try:
        return loop.run_until_complete(coro)
    finally:
        loop.close()

# DEPOIS:
def _run_async(coro):
    """Run async coroutine in a sync Celery task.

    Creates a dedicated event loop per task invocation (required because
    Celery workers are synchronous and asyncpg connections are bound to
    the event loop that created them).

    IMPORTANT: drains all pending asyncpg tasks before closing the loop.
    Without this drain step, asyncpg._terminate_graceful_close() calls
    loop.create_task() on an already-closed loop, raising:
        RuntimeError: Event loop is closed
    which leaves the NullPool session in PendingRollbackError state and
    causes the next task invocation to fail at session.begin().__aexit__.
    """
    loop = asyncio.new_event_loop()
    try:
        return loop.run_until_complete(coro)
    finally:
        try:
            pending = asyncio.all_tasks(loop)
            if pending:
                loop.run_until_complete(
                    asyncio.gather(*pending, return_exceptions=True)
                )
        except Exception:
            pass
        finally:
            loop.close()
```

### Passo 6 — Propor a correção adequada

**Opção A — ENCRYPTION_KEY divergente (mais provável):**
Se a chave no ambiente está errada, a correção é no Cloud Run Console
(não no código) — alinhar `ENCRYPTION_KEY` para o valor original.

**Opção B — Sem suporte a rotação de chave:**
Se `encryption.py` usa `Fernet` simples sem `MultiFernet`,
implementar suporte a múltiplas chaves para permitir rotação segura:

```python
# encryption.py — adicionar suporte a MultiFernet para rotação segura
from cryptography.fernet import Fernet, MultiFernet, InvalidToken
from app.config import settings

def _build_fernet() -> MultiFernet:
    """Build MultiFernet from comma-separated ENCRYPTION_KEY env var.

    Suporta rotação de chave: a primeira chave é usada para encrypt,
    todas as chaves são tentadas em ordem para decrypt.

    Exemplo de rotação:
        Antes: ENCRYPTION_KEY=chave_antiga
        Durante rotação: ENCRYPTION_KEY=chave_nova,chave_antiga
        Após re-encrypt: ENCRYPTION_KEY=chave_nova
    """
    raw = settings.ENCRYPTION_KEY
    keys = [k.strip() for k in raw.split(",") if k.strip()]
    if not keys:
        raise ValueError("ENCRYPTION_KEY não configurada")
    return MultiFernet([Fernet(k.encode() if isinstance(k, str) else k) for k in keys])

_fernet = _build_fernet()

def encrypt(data: str) -> str:
    return _fernet.encrypt(data.encode()).decode()

def decrypt(token: str) -> str:
    return _fernet.decrypt(token.encode() if isinstance(token, str) else token).decode()
```

**Opção C — Dado corrompido no banco:**
Se o dado não tem formato Fernet válido, o usuário precisa re-cadastrar
as credenciais. Neste caso: reportar quais `user_id` estão afetados.
