🔴 2. PERFORMANCE DO ENDPOINT /ohlcv-rate

Você quer:

últimos 60 min (candles/min)

Se fizer algo como:

GROUP BY time

👉 vai gerar cardinalidade alta.

✔ Correto:
SELECT 
  date_trunc('minute', time) as bucket,
  COUNT(*) as candles
FROM ohlcv
WHERE time > NOW() - INTERVAL '60 minutes'
GROUP BY bucket
ORDER BY bucket;

👉 obrigatório usar date_trunc

🔴 3. TRADE PERFORMANCE (RISCO DE N+1)

Você mencionou:

trade real vs simulado

Se fizer join direto com JSON ou múltiplas queries → problema.

✔ Correto:

Use agregação direta:

SELECT
  COUNT(*) as total,
  AVG(pnl_pct) as avg_pnl,
  AVG(holding_seconds) as avg_holding,
  SUM(CASE WHEN outcome = 'WIN' THEN 1 ELSE 0 END)::float / COUNT(*) as win_rate
FROM trade_tracking
WHERE is_simulated = false;
