#225 — CORREÇÃO DO DASHBOARD HEALTH + OBSERVABILITY OPERACIONAL COMPLETA

## CONTEXTO

O dashboard `/dashboard/performance` está apresentando falso negativo operacional.

O sistema REAL está saudável:

- Redis operacional
- Celery Worker operacional
- Celery Beat operacional
- collect_5m operacional
- persistência OHLCV operacional
- 51 símbolos ativos
- ingestão funcionando

Porém o dashboard mostra:

- PARADO
- símbolos = 0
- candles = 0
- delay crítico

A investigação confirmou:

```sql
SELECT
    COUNT(*) AS rows_1h,
    MAX(time) AS last_candle,
    NOW() - MAX(time) AS delay,
    COUNT(DISTINCT symbol) AS symbols
FROM ohlcv
WHERE time > NOW() - INTERVAL '1 hour';

Resultado:

rows_1h > 0
symbols = 51
delay ≈ 8 min

ROOT CAUSE confirmado:
O dashboard assume incorretamente persistência sub-5min, mas o pipeline real opera em:

batch
fila Celery
persistência assíncrona
delay operacional normal ≈ 7–9 min

Portanto:
o erro é EXCLUSIVAMENTE na lógica do dashboard health.
