# CONTEXTO

O sistema possui:

* Filters
* Scoring
* Signals
* Block Rules
* Entry Triggers

As Block Rules têm objetivo de:

* BLOQUEAR trades perigosos
* impedir entradas em condições desfavoráveis
* atuar como camada defensiva

Porém existe uma inconsistência semântica/visual importante entre:

* condição configurada
* resultado matemático
* estado visual PASS/FAIL

Exemplo real do dashboard:

---

TakerRatio < 0.2
Current: 0.3723
Resultado exibido: PASS

VolumeSpike > 4
Current: 0.0500
Resultado exibido: PASS

Spread % > 0.3
Current: 0.0557
Resultado exibido: PASS

VOLUME DELTA < -20
Current: -1003.6
Resultado exibido: FAIL

---

# PROBLEMA

Matematicamente:

0.3723 < 0.2 = FALSE
0.05 > 4 = FALSE
0.0557 > 0.3 = FALSE
-1003 < -20 = TRUE

Ou seja:

* as três primeiras condições são falsas
* a última é verdadeira

Mas o dashboard mostra:

* PASS para as falsas
* FAIL para a verdadeira

Isso gera forte confusão operacional.

# SUSPEITA

A engine aparentemente interpreta:

PASS = regra NÃO acionada
FAIL = regra acionada

Ou seja:

* PASS significa “ativo sobreviveu ao bloqueio”
* FAIL significa “ativo foi bloqueado”

Porém a UI aparenta sugerir:

* PASS = condição verdadeira
* FAIL = condição falsa
