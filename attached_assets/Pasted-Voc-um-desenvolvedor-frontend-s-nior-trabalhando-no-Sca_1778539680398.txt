LAYOUT DA PÁGINA (4 seções)

SEÇÃO 1 — Top bar de métricas (4 cards lado a lado):
  [Saldo USDT disponível] [Capital em posições] [Aprovados ciclo] [Rejeitados ciclo]
  - Polling a cada 10s em GET /api/live/balance
  - Saldo e capital: verde/âmbar respectivamente
  - Contadores do ciclo: reset a cada novo trace_id do sistema

SEÇÃO 2 — Grid 2 colunas:
  COLUNA ESQUERDA: "L3 aprovados — pool ativo"
    - Lista de cards por ativo aprovado (símbolo, score, preço atual, market_type)
    - Score com pill colorida: ≥80=verde · 60-79=âmbar · <60=vermelho
    - Polling GET /api/diagnostics/decisions?status=APPROVED&limit=20
    - Animação fadeInUp no mount, pulse no card quando novo ativo entra

  COLUNA DIREITA: "Posições abertas — aguardando margem"
    - Tabela com colunas: Par · Entrada · Atual · P&L · Margem p/venda · Status
    - Barra de progresso de margem (0-100%): verde ≥70 · âmbar 30-69 · vermelho <30
    - Status labels: "holding" (verde) · "aguardando" (âmbar) · "underwater" (vermelho)
    - Polling GET /api/live/positions a cada 5s
    - Linha piscando levemente se P&L < 0 (animation: pulse 3s)

SEÇÃO 3 — Trilha de execução ao vivo (full width)
  "Trilha de execução ao vivo — spot + futures"
  - Conecta ao SSE: GET /api/live/log-stream
  - Linha de log por evento: [timestamp] [símbolo] [mensagem] [badge de status]
  - Cores por status:
      APPROVED  → borda esquerda #34D399 · fundo rgba(34D399, 0.05)
      REJECTED  → borda esquerda #FBBF24 · fundo rgba(FBBF24, 0.05)
      SKIPPED   → borda esquerda #555B6E · fundo transparente
      ERROR     → borda esquerda #F87171 · fundo rgba(F87171, 0.05)
      SISTEMA   → borda esquerda #4F7BF7 · fundo rgba(4F7BF7, 0.05)
  - Máximo 200 linhas visíveis; scroll automático para o fim ao novo evento
  - Filtros: [Todos] [Spot] [Futures] [Aprovados] [Rejeitados] — chips clicáveis
  - Botão [Pausar] para congelar o scroll sem desconectar o SSE
  - Botão [Limpar] para esvaziar a view local
  - Ao clicar numa linha REJECTED: expand inline mostrando rule_details JSON
    formatado (indicator, value, threshold, operator)

SEÇÃO 4 — Resumo de reprovações (collapsible, fechado por default)
  "Top regras que mais bloquearam — últimas 24h"
  - GET /api/diagnostics/rejections/summary?since=24h
  - Tabela: Regra · Bloqueios · Símbolos afetados
  - Barra de progresso relativa ao maior bloqueador
  - Atualiza a cada 60s
