✔️ Backend — pontos críticos
1) Separar claramente três acumuladores

Hoje o bug nasce da mistura entre “máximo possível” e “penalidade”.

Use:

positive_possible = 0.0   # soma apenas de regras com points > 0
earned_positive = 0.0     # pontos ganhos (regras positivas que passaram)
penalties = 0.0           # soma negativa (regras negativas que passaram)
2) Regra de cálculo correta
for rule in rules:
    pts = rule.points

    if pts > 0:
        positive_possible += pts
        if rule.passed:
            earned_positive += pts
    else:
        if rule.passed:
            penalties += pts  # já é negativo
3) Score da categoria
raw_score = (earned_positive + penalties) / positive_possible if positive_possible > 0 else 0
category_score = max(0.0, min(1.0, raw_score))

👉 Isso resolve:

divisão por zero
cancelamento indevido
penalização real
4) Comportamento desejado (importante validar)
Cenário	Resultado esperado
Só regras positivas	Score normal
Positiva + penalidade leve	Score reduz
Penalidade > positivos	Score = 0
Só regras negativas	Score = 0

Remover + hardcoded é necessário, mas incompleto.

Faça assim:
const formatPoints = (value: number) => {
  if (value > 0) return `+${value}`;
  if (value < 0) return `${value}`;
  return '0';
};
