O problema 3 (SSE OFFLINE) é quase certamente o X-Accel-Buffering: no faltando — esse header é específico do Cloud Run e é a causa mais comum de SSE que nunca entrega eventos. O frontend mostra "tentando reconectar" em loop porque o EventSource abre, o Cloud Run faz buffer do response e nunca flush, e o browser interpreta como timeout.
O problema 2 (L3 vazio) é provavelmente a instrumentação do evaluate_signals.py que ainda não rodou nenhum ciclo desde o deploy, ou o record_decision engolindo exceção de constraint de FK (se o pool_id passado não existir na tabela pools).
O problema 1 (saldo) é provavelmente get_spot_balance() não implementado no gate_adapter.py — esse método foi especificado no prompt da Fase 3 mas o Gate.io v4 exige um GET /spot/accounts assinado com HMAC, e se não estava no adapter original, gerou um AttributeError que retornou 0 silenciosamente.
Rode o curl do diagnóstico 3 primeiro — ele resolve mais rápido e confirma se o backend está vivo antes de depurar os outros dois.
