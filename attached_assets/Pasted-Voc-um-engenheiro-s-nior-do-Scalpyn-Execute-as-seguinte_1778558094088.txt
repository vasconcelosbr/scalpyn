## Tarefa 2 — Health check periódico com alerta

Crie uma nova Celery periodic task chamada `check_structural_coverage` em `backend/app/tasks/health_checks.py` (crie o arquivo se não existir).

A task deve:
1. Executar a seguinte query (ou equivalente via ORM):

```sql
SELECT pc.symbol
FROM pool_coins pc
LEFT JOIN (
    SELECT symbol, MAX(time) AS last_structural
    FROM indicators
    WHERE scheduler_group = 'structural'
    GROUP BY symbol
) i ON pc.symbol = i.symbol
WHERE pc.is_active = true
  AND (i.last_structural IS NULL 
       OR i.last_structural < NOW() - INTERVAL '2 hours')
ORDER BY pc.symbol;
```

2. Se retornar **qualquer símbolo**:
   - Logar em nível WARNING: `f"Structural coverage gap: {len(orphans)} symbols without fresh structural indicators: {orphan_list}"`
   - Disparar `compute_structural.delay(symbol)` para cada símbolo órfão (auto-recovery)
   - Registrar o evento em uma tabela `health_check_log` se ela existir — se não existir, apenas logar, não criar a tabela agora

3. Se não retornar nada: logar INFO `"Structural coverage OK — all active symbols have fresh indicators"`

Regras:
- A task deve usar a sessão de DB assíncrona padrão do projeto (mesmo padrão das outras tasks em `backend/app/tasks/`)
- Registrar no Celery beat schedule com frequência de 30 minutos — localize onde o beat schedule é definido no projeto e adicione lá
- Não criar tabelas novas neste momento
- Adicionar migração Alembic **apenas se** a task precisar de nova coluna em tabela existente
