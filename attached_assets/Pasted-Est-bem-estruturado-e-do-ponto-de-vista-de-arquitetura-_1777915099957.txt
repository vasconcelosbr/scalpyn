⚠️ Ajustes necessários (importantes)
1. Definir explicitamente o que entra no denominador

Hoje você diz:

(sum_matched_points / total_possible_points) × 100

Mas precisa fixar:

👉 total_possible_points inclui TODAS as regras ativas, independentemente de:

match
confiança
disponibilidade de dados

Caso contrário, você cria distorção entre ativos.

Definição correta:

total_possible_points = sum(points for all enabled rules)
2. Tratar ausência de dados (edge case crítico)

Hoje sua task não define isso.

Problema:

indicador não veio / veio null / source UNKNOWN
regra pode cair como False → penalizando injustamente

👉 Isso quebra o ranking.

Solução mínima (sem mudar escopo):

Adicionar no contrato:

Se dado estiver ausente → regra = NOT_MATCHED, mas com flag:
{
  "status": "NOT_MATCHED",
  "confidence": 0.0,
  "data_available": false
}

✔ não muda lógica
✔ mantém consistência
✔ permite evolução futura

3. Garantir que não existe weighting oculto

Você já mencionou, mas precisa deixar explícito:

👉 Proibido qualquer uso indireto de confidence no score

Inclui:

normalização ponderada
filtros antes do score
“pre-adjustments”

Sugestão de regra:

“Confidence MUST NOT influence numerator or denominator in any scoring path.”

4. Contrato do breakdown (padronização)
