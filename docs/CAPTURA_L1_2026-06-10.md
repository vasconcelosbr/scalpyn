## Parte 2 — Implementação

### 2.1 — Ponto de inserção

Arquivo: `backend/app/tasks/pipeline_scan.py`

```python
# Depois de _upsert_assets para L1, antes de continue:
if effective_level == "L1":
    try:
        from ..services.shadow_trade_service import create_l1_spectrum_shadows
        await create_l1_spectrum_shadows(
            user_id=wl.user_id,
            symbols=[a["symbol"] for a in passed],
            execution_id=str(execution_id),
            assets_by_symbol={a["symbol"]: a for a in passed},
            promotion_at=datetime.now(timezone.utc),
        )
    except Exception as _l1cap_exc:
        logger.warning("[PipelineScan] L1_SPECTRUM capture failed (%s) — L3 stream unaffected", ...)
```

**Revisão de pureza:** O bloco acima executa APÓS `_upsert_assets` (apenas gravação do estado L1, sem filtro). `passed` é a lista de símbolos que passaram pelos filtros de perfil — estes são os símbolos em L1. Não há `if` de qualidade entre o `passed` e a chamada de `create_l1_spectrum_shadows`. Os únicos descartes dentro da função são estruturais: hash sampling e reentrada por source.

### 2.2 — Amostragem determinística

```python
_h = int(hashlib.sha256(f"{symbol}:{execution_id}".encode()).hexdigest(), 16) % 10000
if _h < int(sample_rate * 10000):
    # sampled
```

- Reproducível em auditoria (mesmo input → mesma decisão)
- Uniforme (SHA-256, sem correlação com qualidade)
- `execution_id` é constante por ciclo de scan → todos os símbolos no mesmo ciclo são tratados consistentemente

### 2.5 — Constraint por source (migration 073)

```sql
-- Removido:
DROP INDEX ux_shadow_running_user_symbol;  -- (user_id, symbol) WHERE status='RUNNING'

-- Criado:
CREATE UNIQUE INDEX ux_shadow_running_user_source
    ON shadow_trades (user_id, symbol, source)
    WHERE status = 'RUNNING';
```

Impacto no L3 existente: nenhum. L3 shadow running para BTC_USDT não bloqueia L1_SPECTRUM shadow para BTC_USDT e vice-versa.

### 2.6 — shadow_capture_skips

```sql
CREATE TABLE shadow_capture_skips (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id      UUID NOT NULL,
    symbol       VARCHAR NOT NULL,
    promotion_at TIMESTAMPTZ NOT NULL,
    skip_reason  VARCHAR NOT NULL,   -- SAMPLED_OUT | RUNNING_DUPLICATE | RATE_LIMITED
    source_path  VARCHAR NOT NULL,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

- `SAMPLED_OUT`: 1 linha agregada por ciclo (ex: "[24 symbols]") — evita write volume
- `RUNNING_DUPLICATE`: unitário por símbolo (obrigatório para sample weights ML)
- `RATE_LIMITED`: unitário por símbolo

### 2.7 — Rate limit

`shadow_capture_l1_max_per_hour: 200` → 200 shadows/hora (6.4x acima dos ~31/h esperados). Proteção contra surto de promoções L1 em mercado volátil.

```sql
-- 1. Dois streams coexistindo
SELECT source, COUNT(*), MIN(created_at), MAX(created_at)
FROM shadow_trades
WHERE created_at > '<timestamp da ativação>'
GROUP BY source;

-- 2. Taxa empírica ~10% (tolerância ±20%)
SELECT (
    SELECT COUNT(*) FROM shadow_trades WHERE source='L1_SPECTRUM'
      AND created_at > '<ativacao>'
)::float / NULLIF(
    (SELECT COUNT(*) FROM shadow_capture_skips WHERE skip_reason='SAMPLED_OUT'
       AND created_at > '<ativacao>') +
    (SELECT COUNT(*) FROM shadow_trades WHERE source='L1_SPECTRUM'
       AND created_at > '<ativacao>'),
0) AS taxa_empirica;
-- Esperado: ~0.10 (±0.02)

-- 3. Skips registrados
SELECT skip_reason, COUNT(*) FROM shadow_capture_skips
WHERE created_at > '<ativacao>' GROUP BY 1;

-- 4. Instrumentação nos shadows L1 fechados
SELECT COUNT(*) FILTER (WHERE net_return_pct IS NULL) AS sem_net,
       COUNT(*) FILTER (WHERE mae_pct > 0)            AS mae_positivo,
       COUNT(*) FILTER (WHERE atr_pct_at_entry IS NULL) AS sem_atr
FROM shadow_trades WHERE source='L1_SPECTRUM' AND outcome IS NOT NULL;
-- Esperado: sem_net=0, mae_positivo=0, sem_atr pode ter (não crítico)
```
